# 아름다운 이름 - 서비스 요구사항 정의서 (PRD)

본 문서는 **아름다운 이름(소리에너지 성명학 웹 프로그램)** 서비스의 핵심 기능 및 기술적 요구사항을 기록한 정의서(Product Requirements Document)입니다.

---

## 1. 서비스 개요 (Product Overview)
*   **서비스명**: 아름다운 이름 (THE SOUND OF BEING)
*   **목적**: 한글 자음과 모음을 독립적인 소리 파동으로 분류하고 획수로 조율하여 사용자의 삶에 유익한 에너지를 주는 소리에너지 전용 성명학 웹 서비스
*   **대상 사용자**:
    *   **일반 방문자**: 자신의 이름을 분석하여 타고난 파동 성향을 확인하고 전문 상담을 신청하는 유저
    *   **소리에너지 상담사 (관리자)**: 신청된 이름 분석 기록 및 상담 신청서 내역을 조회하고 관리하는 운영자

---

## 2. 핵심 기능 정의 (Key Features)

### 2.1. 이름 분석 기능 (Name Analysis)
*   **입력 양식**: 한글 이름, 실제 생년월일, 날짜 구분(양력/음력), 윤달 여부(음력 선택 시 활성화)
*   **날짜 변환**: 음력 입력 시 `korean-lunar-calendar` 라이브러리를 이용하여 양력 날짜로 자동 변환하여 분석 기준으로 삼음
*   **사운드 에너지 계산**: 자음과 모음의 획수 연산을 거쳐 '나의 핵심 소리수' 및 주기능/부기능 소리 파동수 도출
*   **결과 화면**:
    *   대표 핵심 소리수(1~9번 파동 등) 및 성향 요약 표시
    *   이름 한 글자마다 들어있는 상세 파동수 정보 그리드 노출
    *   상세 아코디언 제공 (성격, 가정, 직업, 여성 특성, 건강, 권장 조언 등)

### 2.2. 컨셉 설명서 (Intro: 이름이 주는 힘)
*   성명학의 본질적 의미와 소리에너지의 필요성을 설명하는 설명글 제공
*   가독성과 세련미를 강조한 텍스트 디자인 적용 (가볍고 화사한 글래스모피즘 박스, 조화로운 포인트 테마 컬러)
*   글 하단에 `나의 이름 분석하기` 행동 촉구(CTA) 버튼 배치

### 2.3. 내비게이션 카테고리 (Category Dropdown)
*   화면 왼쪽 위치에서 드롭다운(Dropdown) 형태로 작동하는 카테고리 메뉴
*   **메뉴 항목**:
    1.  `✨ 나의 이름 분석하기` (메인 분석 폼 이동)
    2.  `📖 이름이 주는 힘` (설명 페이지 이동)
    3.  `📝 상담신청서 작성` (1:1 상담 양식 페이지 이동)

### 2.4. 1:1 상담 신청서 (Consultation Request Form)
*   사용자로부터 상세 상담을 접수하기 위한 폼
*   **요구 입력 항목**:
    1.  신청인 성함 (필수)
    2.  연락처 (필수)
    3.  신청자 이름 (분석 대상자, 필수)
    4.  신청자 생년월일
    5.  가족관계 (예: 2남 1녀 중 장녀)
    6.  직업
    7.  이메일 주소
*   **데이터 흐름**: 작성 완료 후 제출 시 백엔드 데이터베이스에 연동하여 자동 저장 및 신청 완료 피드백 제공

---

## 3. 백엔드 및 보안 요구사항 (Backend & Security)
*   **인프라**: Supabase (BaaS) 활용
*   **보안 아키텍처**:
    *   **인증(Auth)**: 관리자용 이메일/비밀번호 인증 제공
    *   **RLS (Row Level Security)**:
        *   `name_analyses` (분석기록) 및 `consultation_requests` (상담신청) 테이블에 행 수준 보안 적용
        *   일반 방문자(Anonymous): 데이터 **추가(Insert)만 가능**, 다른 사람의 기록 조회 불가능
        *   관리자(Authenticated): 저장된 모든 데이터 **조회(Select), 수정(Update), 삭제(Delete) 권한** 독점

---

## 4. 기술 스택 (Tech Stack)
*   **Frontend**: React.js (Vite 번들러 기반 SPA)
*   **Styling**: Vanilla CSS (CSS 변수 기반 디자인 시스템)
*   **Library**:
    *   `korean-lunar-calendar` (음양력 변환)
    *   `lucide-react` (아이콘 에셋)
    *   `@supabase/supabase-js` (백엔드 클라이언트 SDK)
